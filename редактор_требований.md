давай отвлечемся, и сделаем для этого проекта страницу, в которой из текста будет создана информационная модель требований.
Элементы на странице:
1. Создать модель требований: создает в БД объект "Модель требований": пользователь указывает название модели, система создает объект в БД, и открывает редактор текста.
2. Редактор TipTap, куда пользователь вставляет или пишет текст. Каждый абзац - это потенциальный объект в модели требований, с идентификатором, атрибутами:
  - тип требования: из классификатора бизнес-требование, пользовательское требование, функциональное требование, нефункциональное требование, UX/UI (выбирает пользователь).
